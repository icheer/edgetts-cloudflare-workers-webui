# Impeccable Design Context

## Design Context

### Users
开发者与内容创作者（播客作者、有声书制作者）两类用户并重。使用场景：将长文本批量转成音频，对转换结果做预听和下载。用户熟悉工具型界面，期望高效、不繁琐。

### Brand Personality
**温暖 · 专业 · 克制**
- 温暖：暖橙主色、米白背景，避免冷漠的蓝灰调
- 专业：信息层级清晰，控件行为可预期，无多余装饰
- 克制：不用渐变堆砌，不用图案填充，留白充足

### Aesthetic Direction
- 视觉基调：Notion / Craft 风格，温暖友好，圆角宽松，间距宽裕
- **仅浅色模式**，不跟随系统深色模式
- 主色：赤陶橙 `#c25c18`（降饱和，偏砖红暖调，不刺眼）
- 背景：暖米色渐变 `#fdefd8 → #f8e8d8 → #f0e8f5`，带可见暖色温
- 卡片：全边框 `border: 1px solid var(--border)`，不用单边彩色 accent
- 状态提示：全边框半透明着色，不用 `border-left` 单边强调
- 阴影：带暖棕色调，`rgba(180,100,40,…)`
- 字体：Plus Jakarta Sans（国内用 `fonts.loli.net` 镜像）+ 系统中文字体兜底
- 反参考：不做深色模式、不做单边彩色 border、不做科技感霓虹风、不做彩色渐变标题

### Design Principles
1. **暖色先行** — 主色、强调色、状态色均使用暖色系，避免冷蓝
2. **层级即信息** — 用卡片折叠（details/summary）隐藏次要配置，主流程永远在前
3. **主次按钮明确** — 流式生成（填充主色）为首要操作，标准生成（outline）为次要
4. **无装饰诚实** — 状态提示用全边框半透明着色，图标用线性 SVG 而非 emoji，不用单边彩色 border
5. **响应式连续** — 移动端去掉卡片圆角和阴影，变为全屏直铺，保持阅读舒适度
